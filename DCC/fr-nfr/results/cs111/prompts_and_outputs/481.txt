Key components and relationships:
1. **Client Package**:
   - `MainWindow`: Handles UI interactions and coordinates operations
   - `ClientNetworkHandler`: Manages TCP communication with server
   - `CoordinateGroup`: Domain model with conversion logic

2. **Server Package**:
   - `Server`: Listens for client connections
   - `ServerRequestHandler`: Processes client requests
   - `DatabaseManager`: Manages database operations
   - `CoordinateGroupEntity`: Database entity model

3. **DTO Package**:
   - `CoordinateGroupDTO`: Data Transfer Object for network communication

4. **Key Relationships**:
   - Thick client maintains business logic (coordinate conversions)
   - TCP/IP communication between client and server
   - Clear separation between presentation (Client), business logic (Server), and data storage (DatabaseManager)
   - DTO pattern used for network communication
   - Database entity separate from domain model
