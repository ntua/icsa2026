   - **CoordinateGroupDAOImpl** (Concrete DAO): Implements `ICoordinateGroupDAO`, relying on a `DatabaseConnection` for executing SQL statements.  
   - **DatabaseConnection**: Encapsulates all necessary logic to establish and close connections to a MySQL database.

2. **Controller Package (com.dcc.controller)**  
   - **MainController**: The application’s controller, orchestrating user requests from the view. It interacts with the DAO to perform data operations and uses the `CoordinateConverter` to convert coordinate values, as required by the user’s actions (Create, Update, Delete, etc.).

3. **View Package (com.dcc.view)**  
   - **MainView**: A Swing-based GUI (noted in SRS). It provides forms and buttons for the user to interact with the system, then delegates actions (e.g., create, view, delete) to the `MainController`.  

4. **Relationships**:  
   - The `MainView` holds a reference to `MainController` to send user interactions.  
   - The `MainController` holds references to both the `ICoordinateGroupDAO` (for persistence) and the `CoordinateConverter` (to transform data).  
   - `CoordinateGroupDAOImpl` implements the `ICoordinateGroupDAO` interface and uses `DatabaseConnection` to communicate with the MySQL database.  

This design follows the **MVC** separation of concerns:
- **Model** handles data representation and persistence logic.  
- **View** handles all GUI-related interaction with the user.  
- **Controller** manages the flow between the view and the model components (DAO, conversion, etc.).
